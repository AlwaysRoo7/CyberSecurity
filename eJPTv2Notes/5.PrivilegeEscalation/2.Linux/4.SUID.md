## Find All SUID Files Owned by Root (Potential Privilege Escalation Binaries)
```text
find / -user root -perm -4000 -exec ls -ldb {} \;
find / -perm -4000 -type f 2>/dev/null
```

## SUID Privilege Escalation Explained
In Linux privilege escalation, SUID (Set User ID) is a special type of file permission that allows a user to execute a binary file with the permissions of the file's owner, rather than the user running it. When an executable with the SUID bit is owned by the root user, it runs with root-level privileges regardless of who executes it, making it a primary target for attackers seeking to elevate their privileges from a normal user to root.
