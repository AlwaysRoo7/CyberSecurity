# Nmap Host Discovery Commands

## Default Nmap Scan

Runs a default Nmap scan against the target.

```bash
sudo nmap <ip>
```

---

## Host Discovery

Performs host discovery to identify live hosts on the network.

```bash
nmap -sN 192.168.100.0/24
```

```bash
sudo nmap -sn <ip>/24 -oN discovered_hosts
```

```bash
netdiscover
```

---

## ARP Scan

Performs an ARP-based host discovery scan on the local network and saves the results to a file.

```bash
nmap -sn -PR <T_IP>/24 -oN arp_discovered_hosts
```

---

## Host Discovery Without Port Scanning

Identifies live hosts without performing a port scan.

```bash
sudo nmap -Pn <ip>/24
```

---

## Scan Multiple IP Addresses

Performs host discovery on multiple specified IP addresses.

```bash
sudo nmap -sn 192.168.10.0 192.168.10.1
```

---

## Scan a Range of IP Addresses

Performs host discovery on a specified range of IP addresses.

```bash
sudo nmap -sn 192.168.10.10-254
```

---

## TCP SYN Host Discovery on All Ports

Performs host discovery by sending TCP SYN probes to all ports, helping identify live hosts without performing a full port scan.

```bash
sudo nmap -sn -PS0-65535 192.168.10.0/24
```

### Purpose

- Sends TCP SYN probes to all ports
- Detects live hosts reliably
- Does not perform a full port scan
- Useful when ICMP responses are blocked

---

## Fast Host Discovery Using Common TCP and UDP Ports

Performs a fast host discovery scan using TCP SYN probes to commonly used service ports and UDP probes to NetBIOS ports.

```bash
sudo nmap -sn \
-PS21,22,23,25,53,80,110,139,143,443,445,587,993,995,1433,1521,3306,3389,5900,8080,8443 \
-PU137,138 \
-T4 \
192.168.10.0/24
```

### TCP Ports Used

| Port | Service |
|--------|----------|
| 21 | FTP |
| 22 | SSH |
| 23 | Telnet |
| 25 | SMTP |
| 53 | DNS |
| 80 | HTTP |
| 110 | POP3 |
| 139 | NetBIOS |
| 143 | IMAP |
| 443 | HTTPS |
| 445 | SMB |
| 587 | SMTP Submission |
| 993 | IMAPS |
| 995 | POP3S |
| 1433 | Microsoft SQL Server |
| 1521 | Oracle Database |
| 3306 | MySQL |
| 3389 | RDP |
| 5900 | VNC |
| 8080 | Alternative HTTP |
| 8443 | Alternative HTTPS |

### UDP Ports Used

| Port | Service |
|--------|----------|
| 137 | NetBIOS Name Service |
| 138 | NetBIOS Datagram Service |

### Purpose

- Fast identification of live hosts
- Uses common service ports that are likely to respond
- Combines TCP and UDP discovery methods
- Aggressive timing (`-T4`) for faster results
- Avoids a full port scan while improving host detection accuracy
