# Service Version Detection (`-sV`)

Performs service detection and attempts to identify the version of services running on open ports.

```bash
sudo nmap -sS -sV -sC <ip>
```

### Options Explained

| Option | Description |
|----------|-------------|
| `-sS` | TCP SYN (Stealth) Scan |
| `-sV` | Service and Version Detection |
| `-sC` | Run Default NSE Scripts |

### Purpose

- Identifies running services
- Detects software versions
- Executes default NSE scripts for additional information gathering

---

# Advanced Service Version Detection

Performs a fast, aggressive-timing SYN scan with detailed service and version detection across all TCP ports.

```bash
sudo nmap -T4 -sS -sV --version-intensity 8 -p- <ip>
```

### Options Explained

| Option | Description |
|----------|-------------|
| `-T4` | Aggressive timing template |
| `-sS` | TCP SYN Scan |
| `-sV` | Service Version Detection |
| `--version-intensity 8` | More thorough version detection |
| `-p-` | Scan all TCP ports |

### Version Intensity Levels

```text
0 = Light Detection
1-6 = Moderate Detection
7-9 = Aggressive Detection
```

### Purpose

- Identifies services with greater accuracy
- Detects uncommon applications
- Useful when exact service versions are required
- Slower than default version detection

> Higher version intensity generally improves detection accuracy but increases scan time.

# Aggressive Scan (`-A`)

Performs service detection, OS detection, traceroute, and default NSE script scanning.

```bash
sudo nmap -sS -A -p- -T4 <ip>
```

### Options Explained

| Option | Description |
|----------|-------------|
| `-sS` | TCP SYN Scan |
| `-A` | Aggressive Scan |
| `-p-` | Scan all TCP ports |
| `-T4` | Aggressive timing |

### Includes

- Service Detection (`-sV`)
- OS Detection (`-O`)
- Default NSE Scripts (`-sC`)
- Traceroute

### Purpose

- Comprehensive enumeration
- Operating System fingerprinting
- Service and version identification
- Initial vulnerability assessment

---

# NSE Script Scanning

Runs all NSE (Nmap Scripting Engine) scripts related to FTP services.

```bash
sudo nmap -sS -sV --script=ftp-* -p- -T4 <ip>
```

### Options Explained

| Option | Description |
|----------|-------------|
| `--script=ftp-*` | Run all FTP-related NSE scripts |
| `-sV` | Service Detection |
| `-p-` | Scan all TCP ports |
| `-T4` | Aggressive timing |

### Purpose

- Enumerates FTP services
- Identifies FTP misconfigurations
- Checks for anonymous login
- Detects FTP-related vulnerabilities

### Examples of FTP NSE Scripts

- `ftp-anon`
- `ftp-bounce`
- `ftp-syst`
- `ftp-proftpd-backdoor`
- `ftp-vsftpd-backdoor`

# UDP Service Detection

Performs service and version detection on discovered UDP ports.

```bash
sudo nmap -sV -sU -p 134,177,234 <ip>
```

### Options Explained

| Option | Description |
|----------|-------------|
| `-sV` | Service Version Detection |
| `-sU` | UDP Scan |
| `-p` | Specify UDP ports |

### Purpose

- Identifies services running on UDP ports
- Determines software versions
- Useful for discovering network infrastructure services

### Important Note

When scanning **UDP ports**, always include the `-sU` option. Without it, Nmap will assume the ports are TCP ports and may return inaccurate results.

---

# Common Enumeration Workflow

### 1. Initial Port Scan

```bash
sudo nmap -sS -p- <ip>
```

### 2. Service Detection

```bash
sudo nmap -sS -sV -sC <ip>
```

### 3. Full Enumeration

```bash
sudo nmap -sS -A -p- -T4 <ip>
```

### 4. Targeted NSE Script Scanning

```bash
sudo nmap --script=<service>-* <ip>
```

### 5. UDP Enumeration (If UDP Ports Are Found)

```bash
sudo nmap -sU -sV -p <ports> <ip>
```
