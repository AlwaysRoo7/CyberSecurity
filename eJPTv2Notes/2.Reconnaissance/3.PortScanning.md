## TCP SYN Scan (`-sS`) — Stealth Scan

Performs a TCP SYN scan, commonly known as a **Stealth Scan**.

```bash
nmap -sS -p- <T_IP>
```

### Features

- Fast and efficient
- Does not complete the TCP three-way handshake
- Less likely to be logged than a TCP Connect scan
- Scans all TCP ports (`-p-`)

---

## Skip Host Discovery (`-Pn`)

Treats the target as online and skips the host discovery (ping) phase.

```bash
sudo nmap -Pn <ip>
```

### Use Cases

- When ICMP is blocked
- When the target does not respond to ping requests
- For scanning firewalled hosts

---

## Scan Specific Ports

Scans only the specified ports on the target host.

```bash
sudo nmap -p 80,8080 <ip>
```

### Example Ports

| Port | Service |
|--------|----------|
| 80 | HTTP |
| 8080 | Alternative HTTP |

---

## Scan All TCP Ports

Scans all **65,535 TCP ports** on the target.

```bash
sudo nmap -Pn -p- <ip>
```

### Features

- Comprehensive TCP port scan
- Useful for identifying uncommon services
- Skips host discovery with `-Pn`

---

## TCP Connect Scan (`-sT`)

Performs a full TCP connection scan.

```bash
sudo nmap -Pn -sT <ip>
```

### Features

- Completes the TCP three-way handshake
- Highly reliable
- More easily detected and logged by target systems
- Useful when raw packet privileges are unavailable

### TCP Three-Way Handshake

```text
SYN  → 
SYN/ACK ←
ACK  →
```

---

## UDP Scan (`-sU`)

Scans UDP ports for services such as DNS, SNMP, and DHCP.

```bash
sudo nmap -sU -p 53,123,161 <ip>
```

### Common UDP Ports

| Port | Service |
|--------|----------|
| 53 | DNS |
| 123 | NTP |
| 161 | SNMP |

### Features

- Identifies UDP services
- Slower than TCP scanning
- Useful for discovering network infrastructure services

---

## Fast UDP Scan

Performs a faster UDP scan.

```bash
sudo nmap -sU -F <ip>
```

> **Note:** The correct option is `-F` (Fast Scan), which scans the top 100 most common ports. The `-f` option is used for packet fragmentation, not fast scanning.

### Features

- Scans the top 100 common ports
- Faster than scanning all ports
- Useful for quick reconnaissance

---

## Source Port Spoofing

Sets the source port of outgoing packets.

```bash
sudo nmap -g 53 <T_IP>
```

### Features

- Uses port **53 (DNS)** as the source port
- May bypass poorly configured firewall rules
- Can help evade simple filtering mechanisms

---

## Decoy Scan Using Specific IP Addresses

Masks the real source of the scan by adding decoy IP addresses.

```bash
sudo nmap -D 192.168.1.1,192.168.1.2,ME <T_IP>
```

### Features

- Makes it harder to identify the actual scanner
- Traffic appears to originate from multiple hosts
- `ME` represents the attacker's real IP address

---

## Random Decoy Scan

Uses randomly generated decoy IP addresses.

```bash
sudo nmap -D RND:5 <T_IP>
```

### Features

- Generates 5 random decoy IP addresses
- Obscures the true source of the scan
- Useful for testing detection capabilities

---

# Summary of Common Nmap Scan Types

| Scan Type | Option | Purpose |
|------------|---------|---------|
| SYN Scan | `-sS` | Fast stealth TCP scan |
| TCP Connect Scan | `-sT` | Full TCP handshake scan |
| UDP Scan | `-sU` | Discover UDP services |
| Skip Ping | `-Pn` | Treat host as online |
| Specific Ports | `-p` | Scan selected ports |
| All Ports | `-p-` | Scan all 65,535 TCP ports |
| Fast Scan | `-F` | Scan top 100 ports |
| Source Port Spoofing | `-g` | Use custom source port |
| Decoy Scan | `-D` | Hide source among decoys |
